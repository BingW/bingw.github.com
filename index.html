Structural Functional Annotation

Sep 24th, 2012

Annotaion

  • Structural Annotation - undaries of features of interest (coding regions, regulatory elements, functional RNAs, etc).
    • Ab initio - computationally predicted
    • Comparative - based on similarity to other genes or genomes
    • Experimental - transcript sequencing
  • Functional Annotation - attaching meaning to the features (names, product, activity, biological role, etc.)
    • Sequence homology
    • Structural similarity or structural features
    • Experimental data - gene or protein expression patterns
Errors
  • Structural Annotation Errors
    • Wrong coordinates for the gene (exons, regulatory sites, etc.)
    • False positives - not really a gene (etc.) at all
    • False negatives - true genes (etc.) missing
  • Functional Annotation Errors
    • Overly specific function attributed
    • Wrong function attributed
    • True (known) function missed
    • Typos and misspellings
Quality

  • Model organisms have content rich structural and functional annotations developed and updated over time combining computational predictions with experimental data and manual curation.

  • Non-model organisms can be well annotated, but many are not.

  • Standards for quality and content are still under active development and adherence to the standards is voluntary.

annotation source

  • From an INSD (International Nucleotide Sequence Database) provider
    • Genbank
    • ENA
    • DDBJ
  • From a “Third Party” provider
    • GMOD
    • Other specialized databases

approaches

  • Ab initio
    • Based exclusively on computational models
    • Error prone, esp. for eukaryotes
    • Generally requires manual clean up
  • Comparative
    • Find genes corresponding to sequenced transcripts
    • Find the genes already predicted for a closely related organism

HMMs and annotation

  • Markov models are used in many bioinformatics applications
  • In particular, Hidden Markov Models are used for
    • Ab initio gene prediction
    • Ensemble methods for combining results of multiple gene prediction methods
    • Protein family similarity searches
    • Prediction of other types of features (such as regulatory binding sites)
Using HMMs
  • Given the parameters of the model, compute the probability of a particular output sequence. This problem is solved by the forward algorithm.
  • Given the parameters of the model, find the most likely sequence of hidden states that could have generated a given output sequence. This problem is solved by the Viterbi algorithm.
  • Given an output sequence or a set of such sequences, find the most likely set of state transition and output probabilities. In other words, train the parameters of the HMM given a dataset of sequences. This problem is solved by the BaumWelch algorithm.

HMM introduction

Models and Training Sets
  • The parameters of models are determined from a training set of known positives and negatives, for example a set of known genes, and regions known to not encode genes.
  • Size and quality of training sets impacts the performance of models
  • Size and quality of training sets is related to the specific genome under investigation

The Gene Ontology (GO)

  • Goal = consistent annotation of gene products within and between organisms
  • Gene Ontology Consortium began as a collaboration among model organism dbs (FlyBase, SGD, MGD). Now includes larger number of members and interest groups
  • Ontology = A formal representation of concepts and the relationships among them

Genome Alignment

RNA-seq Goals and Computational Challenges

Goal: + Annotation + Quantification of genes + Quantification of isoforms

Challenges: + Read mapping + Transcriptome reconstruction + Expression quantification

Domain-specific challenges
  • Eukaryotes
    • Processed and unprocessed RNAs in the sample
    • Quality of genome sequences
    • Quality of gene (and alternate splicing) annotations
    • Isoforms
  • Prokaryotes
    • rRNA
    • Gene density
    • Operons
popular in WISC
  • Bowtie, Tophat, Cufflinks, CuffDiff
  • RSEM, EBSeq
Read mapping
  • Unspliced aligners:
    • Seed based (Maq, Stampy)
    • Burrows Wheeler based (BWA, Bowtie, etc.)
    • How similar is your reference genome?
    • Best for mapping against cDNA databases from well characterized reference sequences
    • Can only recognize known exons and isoforms
  • Spliced aligners:
    • Exon first (MapSplice, SpliceMap, TopHat)
      • unspliced alignment, split reads and repeat
      • Most efficient if relatively few reads span junctions
      • Can miss splice sites if reads map elsewhere in their entirety – Seed and Extend (GSNAP, QPALMA)
      • Split all reads into seeds, map, merge and extend
      • Slower, but detect more splice junctions
      • Superior with polymorphic species
Transcriptome Reconstruction Challenges
  • Genes vary in expression level, some may be represented by very few reads
  • Reads can come from either mature mRNA or a precursor
  • When there are multiple isoforms, how can you tell which gave rise to a particular read?
Genome-guided vs. Genome Independent Reconstruction
  • Genome-guided reconstruction
    • Exon identification (G.mor.se)
    • Genome-guided assembly graph (CuffLinks, Scripture)
  • Performance varies with the expression level

  • Genes vary in length - with random fragmentation, long genes are expected to have more reads than short genes expressed at the same level
  • Sequence “runs”/ experiments differ in total number of reads collected
  • RPKM = Reads per kilobase of transcript per million mapped reads (FPKM = fragments…)
  • Many other more sophisticated normalization methods

Variant Discovery Using NGS

Sep 17th, 2012

Types of Variants

  • Simple Single Nucleotide Polymorphisms (SNPs)
  • Multiple adjacent or clustered polymorphisms
  • Insertions and deletions (Indels)
  • Inversions
  • Rearrangements
  • Copy number variation

Separating true variants from artifacts

  • Ignore suspicious regions – areas where too many reads mapped given the expected coverage
  • Require three reads on each strand

Tradeoffs of Sp VS Sn

  • Sn = TP/(TP + FN)
  • Sn = Sensitivity: what portion of the total number of real variants did you detect?
  • Sp = TN/(TN + FP)
  • Sp = Specificity: what portion of the variants you discovered are really variants, not false positives?

Quick_Reference
Syntax

Features of origin Markdown中文

  • Head. use====or----under head.or use#or## or ### or #### … before head
  • Quote. use > infront of quote brock
  • Emphasize. use * or ** or _ or __ wrap text
  • List. use * item,+ item,- item or 1. item,2. item,3. itemstart the line
  • Link. use [example](link) or [example][index] and put [index]:linkin the end
  • Image. same as link, but put ! in the front
  • Code in line. use `code`
  • Code in block. use 4 space in front

Attracting features:

  • Footnote. use [^1] in line and [^1] in the end.
  • Abbreviation. use *[term]: defination in the end.
  • Table. same as rst
  • Span Style. use red{:style:"color:red"} inline behind the term.
  • Code block. use ~~~~ start, use longer ~~~~~~~ end.
  • Horizontal Rules. use * * * * or - - - - or --------
  • LaTeX. use $$ start, $$ end
