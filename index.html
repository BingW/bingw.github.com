Sense From Sequence Reads: Methods for Alignment and Assembly

Paul Flicek & Ewan Birney

Abstruct

review of alignment and assembly algorithms

Background

quality versus cost tradeoff:

  • de novo sequencing: longer reads
  • resequencing: high coverage, paired-end reads
  • mapping TF binding site/location of modified histones: ChIP, short reads
  • DNase-seq: limited to only 20bp of informative sequence

Different error characteristics:

  • Roche 454:insertion or deletion errors during homopolymer runs
  • ABI SOLiD & Illumina/Solexa: errors toward the end of the read

Alignment

Alignment methods(tradeoff between speed and sensitivity):

  • hash table-based implementations
  • Burrows Wheeler transform(BWT)-based methods

Alignment procedure:

  1. heuristic techniques
  2. accurate alignment alogrithms(Smith-Waterman)
Hash-based alignment methods

Sepecially for short reads. methods include:

  • MAQ: From the first 28 bp of the read, MAQ program builds 6 hash tables corresponding to seeds of (length = 8, weight = 4). scans the reference genome against these hash tables. This ensures that all hits with 2 mismatches can be found. 20 hash tables required if get all 3 mismatches.
  • SOPA
  • Illumina’s unpublished ELAND algorithm

These have recently been joined by:

  • SHRiMP: use a q-gram approach, (require mutiple sapced seeds per read
  • ZOOM: manually constructed spaced seeds of weight 14 to enable the detection of up to 4 mismatches in 50-bp reads
  • BFAST
  • MOSAIK: hashes all positions in the reference genome and uses a ‘jump database’ to efficiently locate information in the hash table and thus reduces memory requirements by approximately two-thirds over a naive implementation.
Burrows-Wheeler Transform methods

use the FM index data structure, retains the suffix array’s ability for rapid subsequence search. Much faster than their hash-based counterparts at the same sensitivity level.

  1. the sequence order of the reference genome is modified using the BWT(reversible)
  2. the final index is created; it is then used for rapid read placement on the genome(memory-intensive)

softwares:

  • BOWTIE: Reported 30-fold speed increase over hash-based MAQ
  • BWA
  • SOAP2

Assembly

  1. Sequence at top represents the genome, which is then sampled using shotgun sequencing in base space with 7-bp reads.
  2. the k-mers in the reads (4-mers in this example) are collected into nodes and the coverage at each node is recorded.
  3. he graph is simplified to combine nodes that are associated with the continuous linear stretches into single, larger nodes of various k-mer sizes.
  4. error correction removes the tips and bubbles that result from sequencing errors and creates a final graph structure that accurately and completely describes in the original genome sequence.

softwares:

  • SHARCGS
  • VCAKE
  • VELVET
  • EULER-SR
  • EDENA
  • ABySS
  • ALLPATHS
